# UPower Design — OpenCode Rules

> Place this file as `CLAUDE.md` in your project root to enable UPower commands.

## Slash Commands

### /new [Name]
Initialize a new UPower project from template.
```bash
node ~/.config/opencode/skills/UPower/scaffold/bin/scaffold_project.js --root <cwd> --name [Name]
```
Creates `Source/[Name]/` with standard template structure.

### /build
Execute the manifest-driven build pipeline.
Read `Source/[Name]/project_state.json` → execute the next phase.

### /plan
Show current project state and pending steps.

### /opentalk [Topic]
Start a multi-agent discussion session (Atlas hosts, invites relevant agents).

## Project Structure (after /new)

```
Source/[Name]/
├── input/
│   ├── for_prd/       ← Place raw requirements here
│   ├── prd(input).md  ← Generated by Alice (Product)
│   └── brand_dna.md   ← Generated by Alice (Product)
├── style_prompt.md           ← Bob (Visual)
├── design_system_specs.md    ← Neo (System Architect)
├── animation_prompts.md      ← Bob (Visual)
├── skeleton_template.json    ← Mia (UX)
├── web_content.js            ← Tina (Growth)
└── system_prompt.md          ← Atlas assembles → Ken builds
```

## UPower Config

- **UPower Root**: `~/.config/opencode/skills/UPower/`
- **CLI Entry**: `node <upower_root>/upower.js <command>`
- Never manually edit generated files in `projects/`
